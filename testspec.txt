================================================================================
1. Product overview
================================================================================
VaultRAG is a secure corporate knowledge UI: users ask natural-language questions;
the app retrieves role-filtered chunks and shows answers in a chat layout. Primary
surface is a single-page experience with sidebar history, header role controls,
welcome suggestions, and a bottom input bar.

================================================================================
2. Routes and layout
================================================================================
Main view: `/` (root). Responsive layout: collapsible history sidebar, main chat
stage, fixed input bar at the bottom.

================================================================================
3. UI elements to assert (selectors / visible text)
================================================================================

Sidebar (HistoryPanel):
  - When closed: floating control to open sidebar (aria-label "Open sidebar").
  - When open: "Close sidebar", button "New chat" (Plus + text "New chat").
  - Empty state: "No chats yet", "Your VaultRAG sessions will appear here".
  - With history: section label "Recent"; items show conversation titles; delete
    per row (Trash).

Header:
  - Brand: "VaultRAG" with logo area.
  - Role: chip showing current role text "Employee" or "Executive".
  - Employee mode: button "Unlock Executive" (opens modal flow).
  - Executive mode: button "Use Employee role".
  - Theme toggle with aria-label "Switch to light mode" or "Switch to dark mode".

Welcome state (no messages in active chat):
  - Main heading (h1): "Ask your secure knowledge base".
  - Subtitle mentions ChromaDB, Phi-3, Ollama, Employee vs Executive scope.
  - Exactly four suggestion buttons (pill style), with texts:
      • "What policies apply to my team this quarter?"
      • "Summarize the latest financial figures we have on record."
      • "What security or compliance topics are documented?"
      • "What did we decide about the product roadmap?"

Input bar (InputBar):
  - Backend hint text: "RAG · Phi-3 (Ollama)".
  - TTS toggle (Volume icons; aria-label toggles voice output).
  - Attach: paperclip button, aria-label "Attach PDF or CSV"; disabled when role
    is Employee (`attachDisabled`); enabled in Executive mode.
  - Voice: mic button only if browser supports SpeechRecognition (may be absent in CI).
  - Text input id="search-input", placeholder when not listening:
    "Ask a question about your documents…"; when listening: "Listening…".
  - Submit: button aria-label "Send message" (ArrowUp icon), disabled when input empty.

Chat after messages:
  - User bubbles labeled "You"; system bubbles labeled "VaultRAG".
  - While waiting: typing indicator (three dots) visible when app is in loading state.

================================================================================
4. Functional test cases (priority order)
================================================================================

TC-01 Initial load
  Steps: Open base URL.
  Expect: No uncaught console errors critical to render; welcome state visible;
          heading "Ask your secure knowledge base"; dark/light styling applied;
          input placeholder as specified.

TC-02 Welcome suggestion pills submit the query
  Steps: On welcome screen, click one suggestion pill (repeat for multiple pills
         in separate sessions if needed).
  Expect: Clicking a pill calls send immediately (does not only fill the input):
          user message appears in the thread with that exact text; typing/loading
          indicator may show while the backend responds.

TC-03 Submit via button and Enter
  Steps: Type a non-empty question; click send; new session: type again, press Enter
         (without Shift).
  Expect: Loading/typing state appears; user message in thread; submit clears or
          resets input per app rules; Enter without Shift submits (Shift+Enter may
          be ignored for newline — match InputBar).

TC-04 New chat and history
  Steps: Send a message; open sidebar; click "New chat"; send another message.
  Expect: "Recent" lists entries; active conversation switches; new chat shows
          welcome or empty state per app.

TC-05 Role: Employee vs Executive
  Steps: Default Employee — observe chip "Employee" and Unlock Executive.
         Click "Unlock Executive", complete modal if present, confirm Executive.
         Click "Use Employee role".
  Expect: Chip text and unlock/switch buttons match role; attach availability
          matches role (Executive allows PDF/CSV attach per copy).

TC-06 Theme toggle
  Steps: Toggle theme control twice.
  Expect: aria-label and visible theme-appropriate chrome update.

TC-07 Sidebar open/close
  Steps: Open from float toggle; close via overlay or close control.
  Expect: Panel visibility and accessibility labels as specified.
